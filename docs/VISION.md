# OKO Vision 👁️

**Зачем существует этот проект**

---

# 🧠 1. Основная идея

OKO — это простая система отслеживания ошибок для Python-приложений.

Она должна быть:

* простой в установке
* простой в интеграции
* простой в понимании
* простой в расширении

```python id="v1"
import oko
oko.init()
```

Это единственный гарантированный сценарий использования.

---

# 🎯 2. Проблема, которую решает OKO

Современные системы трекинга ошибок часто:

* слишком сложные для небольших проектов
* требуют внешней инфраструктуры
* перегружены функциями
* долго интегрируются

OKO убирает эту сложность.

---

# ⚙️ 3. Что такое OKO

OKO — это:

* лёгкий event-driven пайплайн
* локальная система трекинга ошибок
* инструмент для разработчиков
* framework-agnostic middleware слой

OKO состоит из двух независимых подсистем:

---

## ⚙️ Processing System (запись и обработка)

Отвечает за обработку событий.

```text
Adapter → Core → Pipeline → Storage + Connectors

```

Функции:

* перехват ошибок
* обработка событий
* сохранение
* отправка уведомлений

### 👁 Observation System (чтение и анализ)

Отвечает за визуализацию и анализ данных.

> Storage → Dashboard

Функции:

* просмотр ошибок
* фильтрация
* статистика
* stack trace analysis

### ⚠️ Важно
> Observation system НЕ влияет на Processing system.
---

# ❌ 4. Что OKO НЕ является

OKO НЕ является:

* распределённой системой логирования
* big data платформой
* enterprise observability системой
* высоконагруженным мониторингом
* полноценной observability платформой уровня enterprise (как Sentry + Datadog)
---

# 👤 5. Для кого этот проект

* solo разработчики
* инди-разработчики
* маленькие стартапы
* MVP проекты
* пет-проекты

---

# 🧭 6. Философия

> Простота — это главная функция.

Внутри система может быть сложной,
но снаружи она всегда остаётся простой.

## 🧭 6.1. Архитектурный принцип разделения

OKO разделяет:

* write path (обработка событий)
* read path (наблюдение)

Это обязательное ограничение архитектуры.

Write ≠ Read

Processing ≠ Observation

---

# 🔒 7. Гарантия стабильности

Публичный API:

```python id="v2"
oko.init()
```

должен оставаться:

* стабильным
* минимальным
* обратно совместимым

---

# 🚀 8. Долгосрочное развитие

OKO может развиваться внутри как:

* система плагинов
* расширенный event pipeline
* новые коннекторы (Telegram, Webhook и т.д.)

Но принцип останется:

> минимум конфигурации всегда
