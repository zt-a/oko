
# OKO Roadmap 👁️

---

# 🧱 Этап 1 — MVP (ядро системы)

Цель: рабочая система трекинга ошибок с визуальным контролем

---

## ⚙️ Core система

* [ ] модель Event
* [ ] Core: очередь + worker
* [ ] pipeline (минимальный)
* [ ] FastAPI адаптер
* [ ] WSGI адаптер (Flask/Django)
* [ ] Telegram коннектор
* [ ] SQLite хранилище
* [ ] интеграция logging / loguru

---

## 👁️ Dashboard (MVP ОБЯЗАТЕЛЬНО)

Цель: базовый локальный интерфейс наблюдения

* [ ] локальный web dashboard (FastAPI / simple HTTP server)
* [ ] список ошибок
* [ ] просмотр stack trace
* [ ] базовые фильтры
* [ ] endpoint `/errors`
* [ ] endpoint `/error/{id}`

```text
Storage → Dashboard (read-only)
````

📌 Dashboard — обязательная часть продукта, не опциональный модуль.

---

## 📦 MVP результат

После этапа 1 пользователь должен иметь:

* pip install oko
* oko.init()
* Telegram уведомления
* локальный dashboard в браузере

---

# ⚙️ Этап 2 — стабильность

Цель: сделать систему безопасной для продакшена

* [ ] batching событий
* [ ] rate limiting Telegram
* [ ] дедупликация ошибок
* [ ] структурированный логинг
* [ ] корректное завершение worker
* [ ] оптимизация SQLite записи
* [ ] оптимизация dashboard запросов

---

# 🧠 Этап 3 — “умная” обработка

Цель: улучшение качества данных

* [ ] группировка одинаковых ошибок (advanced)
* [ ] fingerprinting ошибок (hash stack trace)
* [ ] обогащение контекста (request info)
* [ ] классификация ошибок по важности
* [ ] фильтрация событий
* [ ] улучшение dashboard аналитики

---

# 🌐 Этап 4 — экосистема расширений

Цель: plugin-based архитектура

* [ ] API для кастомных connectors
* [ ] API для кастомного storage
* [ ] API для адаптеров
* [ ] система плагинов
* [ ] plugin API для dashboard расширений

---

# 📊 Этап 5 — расширенный dashboard (UI evolution)

Цель: сделать dashboard полноценным аналитическим интерфейсом

* [ ] UI с графиками
* [ ] таймлайн ошибок
* [ ] поиск по логам
* [ ] фильтры по проектам
* [ ] aggregation metrics (error rate, spikes)
* [ ] performance view

---

# 🚫 Не цели проекта

* распределённая очередь
* enterprise observability система
* тяжелая cloud-инфраструктура
* Kubernetes-native monitoring

---

# 🧭 Финальная философия

> OKO — это локальный инструмент наблюдаемости для разработчика, а не инфраструктурная платформа.
